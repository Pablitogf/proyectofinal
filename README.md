## Descripción del Proyecto

Este proyecto implementa un modelo de dominio para la gestión de solicitudes académicas dentro de una institución universitaria.

El sistema permite que los estudiantes creen solicitudes que posteriormente pueden ser clasificadas, atendidas y cerradas por coordinadores o personal administrativo.

El diseño del sistema está basado en principios de **Domain Driven Design (DDD)**, separando claramente las responsabilidades del dominio mediante:

- Entidades
- Value Objects
- Agregado raíz
- Reglas de negocio
- Pruebas unitarias

El objetivo principal es modelar correctamente el dominio del problema garantizando la consistencia de las reglas del negocio.
