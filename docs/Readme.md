Descripción breve del proyecto basada en el contexto institucional Proyecto backend para una institución universitaria que modela la gestión de solicitudes académicas. Los estudiantes registran solicitudes y, aplicando reglas del dominio, el sistema permite que coordinadores/personal responsable las clasifiquen, asignen, atiendan y cierren, siguiendo un ciclo de vida controlado. El diseño está basado en DDD (Agregado raíz Solicitud, entidades como Usuario, value objects como CodigoSolicitud/Email, y excepciones de dominio).
